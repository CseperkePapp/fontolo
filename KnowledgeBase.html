Fontolo CSS Knowledge Base

A complete reference for the Fontolo CSS architecture, file by file.

Purpose

This file establishes the foundational styles for the entire system. Its primary job is to create a consistent "reset" across all browsers by removing default margins and setting base font properties. This ensures that the layout system has full and predictable control over all spacing.

Purpose

This is the most important file for controlling space and structure. It contains all the layout containers that manage the positioning of components, such as .stack, .grid, and .flex. It also defines the responsive behavior of the main application layout.

Purpose

This file is a central library for all the primary UI building blocks. It defines the self-contained appearance of each component (padding, colors, borders, etc.) while strictly following the "Inside vs. Outside" rule, meaning no component has any external margins.

Detailed Breakdown

Header Components: Includes .app-header and related classes that structure the main application header.

Button Components: Defines the base .btn class and its color and size modifiers (e.g., .btn-primary, .btn-sm).

Card Components: Contains the .card component and its internal sections like .card-header.

Form Components: Styles all form elements, including inputs, labels, sliders, and custom selects for a consistent look.

Accordion Components: Defines the styles for the .accordion-item and its interactive parts, including the expand/collapse animation.

Purpose

This file handles contextual styling. It contains rules that change a component's appearance based on where it's placed. For example, it makes typography smaller when inside a .panel or .sidebar. It should be loaded late in the CSS order to ensure its rules take precedence.

Purpose

This file contains styles that are specific to the live configurator demo page. It includes classes for the color swatches, spacing scale demonstrators, and other UI elements that are used for showcasing the design system rather than being part of the core component library.
